Sequtus runs on a simple model



  • The main program creates an Application Core
  • This creates one or more Screens
  • Battle Screens are a subclass of Screens for use as part of the core "game", these handle player IO input (keyboard and mouse)
  • Battle Screens all have a Simulator which is responsible for the processing side of things
  • Simulators spawn AI processes to run any computer teams they are responsible for
  • Simulators also connect to a Game Server to allow network play
